# correctness/

Papers for when model produces errors, inconsistent answers, or unverified claims.

## Subdirectories

| Directory       | What                                    | When to read                       |
| --------------- | --------------------------------------- | ---------------------------------- |
| `sampling/`     | Generate multiple candidates, aggregate | Model gives inconsistent answers   |
| `verification/` | Explicit checking steps catch errors    | Model produces unverified claims   |
| `refinement/`   | Feedback loops improve output           | Output needs iterative improvement |
